# Binja-NO-MCP

将 Binary Ninja 的当前分析状态导出为可供 AI Agent 离线分析的稳定工件集合。

## 术语

**导出快照**：
一次导出会话产生、供 AI Agent 分析二进制的稳定工件集合；它代表一个特定时刻的 Binary Ninja 分析状态。
_避免使用_：导出目录、文本转储

**完整快照**：
导出开始时冻结的全部已识别函数均已完成分析并可被索引的导出快照。启动入口仅影响 Agent 的阅读起点，不限制快照内容。取消、解析失败或分析未就绪的结果属于部分结果。
_避免使用_：成功导出

**当前快照**：
某个目标当前保留的一份导出快照。其完整性必须由快照头判断，不能由目录存在或导出时间推断。全量导出直接替换旧快照；增量导出在同一目标的已有快照中保留已成功的函数工件，并重试其余函数。
_避免使用_：快照历史、差异基线

**快照头**：
`meta/binary.json`，当前快照的唯一入口元数据，声明 schema、完整性状态和函数计数。Agent 必须先读取它。开放式程序分析随后从启动入口清单开始；用户明确指定函数时定向查询函数索引；纯元数据查询直接读取对应元数据工件。
_避免使用_：独立 manifest、多个入口元数据

**分析引用**：
Binary Ninja 分析报告的源到目标关系，用于导航但不代表完整事实；未报告某条边不表示该关系不存在。
_避免使用_：完整 xref、已验证调用

**可验证直接调用**：
调用目标可从函数 IL 中直接识别的调用关系。它不涵盖运行时计算出的间接调用。
_避免使用_：推测调用、完整调用图

**调用证据**：
存放在源函数 meta 中的调用关系记录。可验证直接调用记录调用点与目标函数标识；间接调用明确记录为目标未知。函数索引不承载这些边。
_避免使用_：全局 xref 文件、无来源调用图

**函数标识**：
单架构导出快照内以未补零十六进制函数起始地址表示的稳定标识，例如 `0x88504`。它用于索引和元数据；函数工件以经文件名清理后的函数名称命名，未命名函数使用 `sub_<offset>`。
_避免使用_：以文件路径替代稳定函数标识

**函数索引**：
`meta/function_index.jsonl`，每个函数一行的查找目录，包含函数标识、名称、声明、可读工件路径、导出状态及适用的启动阶段。它不承载 xref 列表或完整函数元数据。
_避免使用_：第二个搜索库、调用图索引

**带地址反编译文本**：
HLIL 和 pseudo-C 的每个已关联地址的行以前导、去除前导零的小写十六进制虚拟地址输出，随后保留反编译文本。地址列只用空格对齐，不补零；没有关联地址的行保留空地址列，不复制相邻地址。
_避免使用_：独立行映射、推测行地址

**目标二进制**：
运行于 Android 的 AArch64 ELF 二进制，包括可执行文件、PIE 和共享库。启动定位与入口发现只需保证这一范围。
_避免使用_：仅 `.so`、通用 ELF、多架构二进制

**启动入口清单**：
`meta/startup_entries.json` 中按启动阶段标记的已确认启动地址清单。数组槽位保持原始顺序和重复项；地址即使未被 Binary Ninja 识别为函数也保留，此时它没有函数索引记录或函数工件。它用于指导 Agent 按需展开分析，不包含预先计算的可达函数切片，也不代表完整的运行时执行轨迹。
_避免使用_：函数执行顺序、完整执行路径、启动切片

**启动路线**：
针对一种 Android ELF 目标类别生成的、带阶段标签的入口顺序。它只列出可确认且适用的入口，不以单一路线覆盖所有二进制。
_避免使用_：固定入口序列、全局执行顺序

**已确认的 main**：
Android AArch64 可执行文件中由符号或标准 bionic CRT 直接传递给 `__libc_init` 的 main 函数。其他候选不作为启动入口输出。
_避免使用_：启发式 main、推测 main

**剥离目标**：
缺少可依赖符号或节名的 Android AArch64 ELF 目标。其启动入口必须从运行时动态结构识别，节信息仅可作为辅助线索。
_避免使用_：带符号调试目标、节名可信目标

**段记录**：
`meta/segments.json` 中的 Binary Ninja 映射范围，声明虚拟地址边界、读写执行权限及原始文件支撑范围。它用于地址归属和原始文件偏移换算，不是原始 ELF program header 的逐字复制，也不包含段字节。
_避免使用_：段转储、原始 program header

**节记录**：
`meta/sections.json` 中的 Binary Ninja 节语义记录，声明名称、范围、语义、类型及对齐信息。它用于语义导航；对剥离目标，节名是辅助证据，不能替代动态启动结构。
_避免使用_：节字节、启动入口真值

**原始字节证据**：
因导出快照未包含目标数据字节而通过段记录映射、从原始二进制读取的字节。它只覆盖文件支撑范围，可能不含 Binary Ninja 映射视图中已经应用的重定位，必须与导出快照证据分开陈述。
_避免使用_：BinaryView 映射值、已重定位数据

**JNI 加载入口**：
可识别的 `JNI_OnLoad` 函数，是 Android 原生库的启动入口清单项。已注册 native 方法由 Agent 从该入口按需追踪，导出器不预先恢复映射。
_避免使用_：JNI API 映射、全部 JNI 方法索引
